
# 컨벤션
# build   : 빌드 시스템 또는 외부 환경에 변화를 주는 커밋(gulp, broccoli, npm)
# ci      : 배포 관련 설정 파일의 변경 커밋
# fix     : 버그를 수정하는 커밋
# feat    : 새로운 기능을 추가하는 커밋
# docs    : 문서를 추가하거나 수정하는 커밋
# test    : 단위 테스트를 추가하거나 수정하는 커밋
# perf    : 기능 변경없이 성능을 향상 시키는 커밋
# refactor: 버그 수정이나 기능 추가 없는 코드 변경 커밋
# style   : 코드의 의미에 영향을 주지 않는 코드 변경 커밋(공백, 코드 포맷, 세미콜론 수정, 콘솔로그 제거 등)
# chore   : 이외의 모든 잡다한 커밋. 만약 여러 커밋에 걸쳐 하나의 기능 추가를 한다면 하나의 커밋에 feat를 지정하고 나머지 커밋은 chore 를 지정
